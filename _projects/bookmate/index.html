The first week of our project was primarily about user research. Given that Bookmate did not build off a previous capstone project, we conducted some informal interviews that provided us some insight into the current strategies UW students use to exchange used textbooks with one another. With this information, we learned what aspects of the textbook transaction process are important to UW students and translated them data and functional requirements that our application would support. We then organized these requirements into a more concise information architecture through affinity diagramming and designed a corresponding system map that conveyed the flow of our application.

Basically what we did this week was:
1. Establish a paper prototype by comparing and combining various interface sketches
2. Test that prototype with users
3. Make changes according to feedback
4. Test the updated design with users
5. Establish a final prototype design
6. Start working on an interactive wireframe and digital mockups (static) to increase the fidelity of our design and prepare for further testing. It's also important to note that while we have established the initial search and results parts of the interface, the buyer/seller interaction elements need further refinement before they can be integrated into the wireframe and mockups. We hope to receive feedback and refine these by Tuesday.

Things we have done this week:

3. Created digital mockups of the initial screens
4. Designed the initial database schemas
5. Setup the web server
